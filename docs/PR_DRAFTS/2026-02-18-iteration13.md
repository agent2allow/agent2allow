# [SEC] Add optional API key auth for approval mutation endpoints

## Problem
- Approval actions had RBAC checks but no dedicated endpoint authentication mechanism.

## Solution
- Added `ApprovalApiKeyAuth` with optional enablement:
  - `APPROVAL_API_KEY_ENABLED=true`
  - `APPROVAL_API_KEYS='{\"k-ops\":\"ops-reviewer\"}'`
- Enforced API key checks on:
  - `POST /v1/approvals/{id}/approve`
  - `POST /v1/approvals/{id}/deny`
  - `POST /v1/approvals/bulk`
- Mapped valid key identity into approver attribution for RBAC and audit consistency.
- Extended integration/unit tests and updated SDK/docs usage patterns.

## How to test
```bash
cd gateway && pytest -q tests/test_api_auth.py tests/test_integration.py
```

## Breaking changes
- None (feature is opt-in and disabled by default).
